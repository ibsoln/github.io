    Configuration Changes

    This section identifies the changes to the Sync Gateway Configuration Schema for Release 2.8

    Structural Schema Changes

    _replicate becomes database.replication

    The replication configuration property _replicate is deprecated. Inter-sync-gateway Replication (v2) replications are now defined with the _replication property, which is as a member of the database property. Note that any given database can still have multiple replications configured.

    Constraint: Inter-sync-gateway Replication (v2) does not support replication between two remote hosts via a third Sync Gateway.

    Removed or Deprecated Properties

    The following SG Replicate 1.0 (SGR-1) ReplicationConfig configuration properties are deprecated. Deprecated functionality remains solely for backward compatibility with SGR-1 but is no longer supported:

    create_target

    SG 1.0 Deprecated. The create_target property remains in ReplicationConfig but is deprecated for Couchbase Server-backed use cases.

    proxy

    SG 1.0 Deprecated. The proxy property remains in ReplicationConfig but is deprecated_.

    doc_ids

    SG 1.0 Deprecated. The doc_ids property remains in ReplicationConfig but is deprecated_.

    source

    SG 1.0 Deprecated.

    For SGR-2: the source property is, implicitly, the database associated with the replication (that is, the 'database' property under which it is defined).

    target

    SG 1.0 Deprecated.

    For SGR-2: The target property is replaced by the remote property.

    async

    Removed - redundant.

    The async property is a REST-API-only property. It was formerly the method used to force a request to start without awaiting completion of one-shot replications (that is, to run asynchronuously). Now, it always run in async mode.

    changes_feed_limit

    Removed - redundant.

    New Configuration Properties

    The following new properties are added:

    remote

    The remote property replaces the former target.

    The remote property represents a database URL for the remote Sync Gateway. To use another database hosted on this Sync Gateway, specify its full Sync Gateway database URL here.

    direction

    The direction property indicates whether the replication is pushed or pulled by this node. Valid values are: push, pull, and push_and_pull.

    conflict_resolution_type

    The conflict_resolution_type property defines how conflicts are resolved.

    Valid values are: default, localWins, remoteWins, custom

    custom_conflict_resolver

    The custom_conflict_resolver property specifies the custom Javascript function to be used for conflict resolution. It is valid only when conflict_resolution_type = custom.

    purge_on_removal

    Boolean value for whether channel removals should trigger purge Defaults to false

    enable_delta_sync

    Boolean value for whether to enable delta sync for this replication.

    maximum_backoff_time

    The maximum_backoff_time property deines the period of time during which Sync Gateway will attempt to reconnect unreachable targets.

    Replication state

    The replication_state property reports the current state of a replication.

    Possible values in replicationStatus response bodies are: stopping, stopped, starting, active.

    Valid values in configuration or in a payload to a replication endpoint are: stopping, starting.

    Modified Configuration Properties

    The following replication configuration properties have changed:

    cancel

    This property requests a running replication stop. It is valid only when using ReplicationConfig for _replicate REST API calls.

    If defined in the configuration schema, it will log an error and prevent start up.

    When used in the REST API, you must specify the replication_id of a running replication or the request will return an error.

    Retained Configuration Properties

    The following replication configuration properties are retained unchanged:

    • continuous

    • filter

    • query_params

    Constraints

    • In order to support Inter-sync-gateway Replication (v2), all active nodes in active cluster must have min release version of Hydrogen.

      Documentation: Need to call this out

    • Sync Gateway (Hydrogen) will only support 2.x versions of SG-Replicate.

      When SGR-1 replication is between two remote databases

      Replication between two remote databases is no longer supported.

      Sync Gateway will log an error and fail to start if a replication defined at the configuration’s root level does not involve at least one local database (source or target).

      When SGR-2 is targeting a pre-2.8 Sync Gateway

      • Push replications do not support a pre-2.8 target with "allow_conflicts": false set.

        The target must have that configuration set to true for a replication to work.

      • Push replications do not use Delta Sync when pushing to a pre-2.8 target

    Deprecation Notices

    DNSGW-2.8/001 Inter-sync-gateway Replication (v1) replication protocol

    Inter-sync-gateway Replication (v1) (SGR-1) is deprecated in Sync Gateway versions starting with version 2.8. You should plan your transition to SGR-2 now to avoid potential issues when this functionality is removed.

    The functionality of Inter-sync-gateway Replication (v1) remains unchanged, unless explicitly stated in these release notes and-or in the appropriate documentation section. Specifically, replications are scheduled on the node on which they are created.

    Refer to Notes on Upgrading for more information on moving from SGR-1 to SGR-2.

    Configuration deprecations

    The SGR-1 replication configuration method is deprecated at version 2.8.

    Whilst replications configured at the root level will continue to function, you should configure new replications under the database in the databases.{dbname}.replications.{replication_id} property.

    REST API Deprecations

    These SGR-1 replication REST endpoints are deprecated:

    • active_tasks — which is superseded by new monitoring endpoint

    • _replicate which is superseded by new replication endpoint
